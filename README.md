# 🍕 PizzaShop API

**REST API для управления заказами в пиццерии** — MVP с аутентификацией, эмуляцией платежей, асинхронной обработкой событий и интеграционными тестами.

> 🎯 Проект создан в учебных целях для отработки: транзакций, многопоточности, событийной модели и тестирования с реальной БД.
>
> ⚠️ **Статус проекта**: В разработке. Акцент делался на работу с асинхронностью и транзакциями, поэтому некоторые базовые CRUD-операции ещё не реализованы.

## 🔧 Требования
- Java 21+
- Docker (опционально, для БД)
- Gradle (или используйте `./gradlew`)

## Ключевые возможности

### 👥 Для клиентов

- Регистрация и JWT-аутентификация
- Просмотр меню с фильтрацией
- Корзина (добавление/удаление/очистка)
- Оформление заказа с эмуляцией оплаты

### Для администраторов

- CRUD для ингредиентов
- CRUD для пицц
- CRUD для пользователей (ещё не реализовано)

## Технические особенности
### 🔐 Безопасность

- JWT-аутентификация с разделением ролей (USER/ADMIN)
- Фильтр JwtTokenFilter с whitelist'ом публичных эндпоинтов
- PasswordEncoder на BCrypt

### 📦 Транзакции и конкурентность

- Оптимистичные блокировки (@Version) для ингредиентов — защита от одновременного изменения цены
- REQUIRES_NEW propagation в KitchenService.acceptPaidOrder() — гарантирует обработку заказа на кухне даже при откате основной транзакции

### ⚡ Асинхронность и события

- @TransactionalEventListener — пересчёт цен пицц происходит только после успешного коммита изменения цены ингредиента

- #### Два настроенных пула потоков:
``` java
  priceRecalculationExecutor (core=3, max=5, CallerRunsPolicy) // для пересчёта цен

  paymentProcessingExecutor (core=10, max=40, DiscardOldestPolicy) // для обработки платежей
```

### 💳 Эмуляция платежей

  Реализован модуль PaymentService с:

  - Имитацией сетевых задержек (2-5 сек)
  - Разными сценариями ответов (70% успех / 30% неудача)
  - Асинхронной обработкой с CompletableFuture

### 🧪 Тестирование

- Unit-тесты сервисов с Mockito

- Интеграционные тесты с Testcontainers:

    - Поднимается реальный PostgreSQL в Docker
    - Тестируются контроллеры и репозитории
    - Проверяется работа событийной модели

### 📊 Работа с БД

- Hibernate/JPA с кастомными запросами
- Проверка использования ингредиентов перед удалением (countPizzaUses)
- Пагинация для списков (ингредиенты, пиццы)
- ФИльтрация с использованием Criteria API
- DTO-проекции для оптимизации запросов (PizzaCardProjection)

## 🚀 Быстрый старт

```bash
# 1. Клонировать репозиторий
git clone https://github.com/Egrius/pizzaShop.git
cd pizzaShop

# 2. Запуск PostgreSQL (если есть Docker)
docker run --name pizzashop-db -e POSTGRES_DB=pizzashop -e POSTGRES_USER=admin -e POSTGRES_PASSWORD=admin -p 5432:5432 -d postgres:16

# 3. Сборка и запуск
./gradlew bootRun
```
